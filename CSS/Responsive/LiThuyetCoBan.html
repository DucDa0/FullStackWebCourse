Media query(@media) /--- syntax ---/ @media not|only mediatype and (mediafeature and|or|not mediafeature){ css code; } /--- key word ---/ not: bỏ chọn 1 đối tượng nào đó only(default): chỉ áp dụng lên đối tượng nào đó and: và or: hoặc /--- mediatype ---/ print: áp dụng cho tính năng in. screen: áp dụng chế độ màn hình(tv, desktop, phone,..) speech: màn hình ns đc, dành cho ng khuyết tật. all(giá trị mặc định): áp dụng tất cả 3 cái trên. /--- mediafeature --/ min-width: lớn hơn hoặc = max-width: nhỏ hơn hoặc = /--- poly fill ---/ Được sử dụng để giải quyết 1 số trường hợp trình duyệt ko hỗ trợ media /--- ví dụ ---/ @media only screen and (min-width: 960px){// nếu mà width lớn hơn = 960px thì code sẽ đc thực thi span{ display: block; } } 2. Grid System(*Nền tảng rất quan trọng*) + Column-Cột: độ rọng đơn vị %(tương đối) giúp linh động, dễ dàng tương thích với độ rộng khác nhau của các thiết bị. Số lượng cột trong GS đc xác định trước. Ví dụ: PC 12|16 cột, tablet 8 cột, mobile 4 cột + Gutter-Đường ngăn cách(rãnh ngăn): là khoảng cách 2 phía của 1 cột, tạo nên rãnh giữa các cột. độ rộng rãnh ngăn có thể thay đổi cho phù hợp với thiết kế hoặc độ rộng màn hình VD: PC/Tablet 24pz, mobile 16px + Margin-Phần lề Là khoảng cách 2 bên trái\phải của bố cục màn hình chính của website. Độ rộng phần lề thay đổi để phù hợp với các kích thước màn hình. VD: Phần lề lớn thích hợp cho màn hình lớn như PC, phần lề nhỏ thích họp cho màn hình nhỏ Tablet , mobile 6. Thành phần chính(làm vc với css) a.
